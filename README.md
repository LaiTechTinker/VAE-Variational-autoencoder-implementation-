# What is a Variational Autoencoder?

A **Variational Autoencoder (VAE)** is a generative deep learning model that learns the **underlying probability distribution of data** so that new samples can be generated from it.

Unlike a standard autoencoder that compresses an input into a deterministic vector, a VAE learns the **parameters of a probability distribution** representing the latent space.

Instead of encoding an image into a single vector, the encoder outputs:

* **Mean (μ)**
* **Log Variance (log σ²)**

These parameters define a **Gaussian distribution** in the latent space.

From this distribution we sample a latent vector **z**, which is then passed to the decoder to reconstruct the image.

---

# VAE Architecture

The VAE consists of two main neural networks:

### Encoder

The encoder compresses the input image into two vectors:

* mean (μ)
* log variance (log σ²)

These represent the parameters of a Gaussian distribution.

Input image → Encoder → μ , log σ²

---

### Reparameterization Trick

Direct sampling from a distribution blocks gradient flow during backpropagation. To solve this, VAEs use the **reparameterization trick**:

z = μ + σ × ε

Where:

ε ~ Normal(0,1)
σ = exp(0.5 × logvar)

This allows the network to remain **fully differentiable** during training.

---

### Decoder

The decoder takes the sampled latent vector **z** and reconstructs the image.

Latent vector z → Decoder → Reconstructed Image

---

# Loss Function

Training a VAE involves optimizing two losses.

### 1. Reconstruction Loss

This measures how well the reconstructed image matches the original image.

Common choices:

* Mean Squared Error (MSE)
* Binary Cross Entropy (BCE)

Example:

reconstruction_loss = MSE(x_reconstructed, x_original)

---

### 2. KL Divergence Loss

The KL Divergence forces the learned latent distribution to stay close to a **standard normal distribution N(0,1)**.

KL formula:

KL = -0.5 × Σ (1 + logσ² − μ² − exp(logσ²))

This ensures the latent space remains **continuous and smooth**, which allows meaningful sampling.

---

### Total VAE Loss

Total Loss = Reconstruction Loss + KL Divergence

This balances **reconstruction accuracy** with **latent space regularization**.

# Training

The training loop performs the following steps:

1. Pass input images through the encoder
2. Obtain μ and log variance
3. Sample latent vector using reparameterization
4. Pass latent vector to the decoder
5. Compute reconstruction loss
6. Compute KL divergence
7. Backpropagate and update parameters

Example training logic:

for epoch in range(epochs):

```
total_loss = 0

for images, _ in train_loader:

    images = images.to(device)

    recon_images, mu, logvar = model(images)

    loss = vae_loss(recon_images, images, mu, logvar)

    optimizer.zero_grad()
    loss.backward()
    optimizer.step()

    total_loss += loss.item()
```

# Generating Images

After training, new images can be generated by **sampling random vectors from the latent space**.

Example:

z = torch.randn(64, latent_dim).to(device)

with torch.no_grad():

```
samples = model.decoder_input(z)
samples = samples.view(-1,128,4,4)
samples = model.decoder(samples)
```

Generated samples can then be saved using:

torchvision.utils.save_image(samples, "generated_samples.png")

# Latent Space Intuition

Because the latent space is regularized to follow a **Gaussian distribution**, it becomes smooth and structured.

This allows:

* interpolation between images
* sampling new images
* learning meaningful representations

Nearby points in latent space correspond to **similar images**.
